# Codex 懶人包 #01.5：建立第一個 Codex 專案工作區

> 版本：v0.2（Codex Desktop 版）
> 更新日期：2026-04-27
> 對應影片：Codex基本功 EP02（建議主題）

> 本懶人包接在 #00 環境建置之後。目標是讓新手建立第一個真正可工作的 Codex 專案資料夾，補齊 `AGENTS.md`、`README.md`、`.gitignore`，並順便檢查初學者必裝的 Skills 與 Plugins。

---

## 這個懶人包會幫你做什麼？

完成後，你會有一個乾淨的 Codex 專案工作區：

- 有一個固定資料夾作為工作區
- 有 `AGENTS.md` 告訴 Codex 這個專案怎麼工作
- 有 `README.md` 給人看的專案說明
- 有 `.gitignore` 防止 API Key、暫存檔被放進 GitHub
- 可驗證 Codex 能在這個資料夾裡讀檔、寫檔
- 可選擇是否初始化 Git，但不會自動 push
- 會檢查初學者必裝的 Skills 與 Plugins

---

## 先備條件

- [ ] 已完成 [[00-環境建置]]
- [ ] Codex Desktop app 已登入
- [ ] 你知道要把專案放在哪個雲端同步資料夾

建議位置：

| 情境 | 建議位置 |
|------|----------|
| 一般教材 / 程式專案 | Google Drive：`G:\我的雲端硬碟\<專案名稱>` |
| 個人筆記 / 第二大腦 | Obsidian vault：`C:\Users\<你>\OneDrive\文件\Secondbrain` |
| 暫時測試 | `Documents\CodexTest` |

> [!tip]
> 影片示範可用 Google Drive 建立 `2026Codex`。重點不是一定要叫這個名字，而是要放在你之後找得到、也能跨電腦同步的位置。

---

## 請 Codex 幫我執行以下步驟

> [!warning]
> 以下是給 Codex 讀的操作流程。新手如果還沒決定專案名稱，Codex 要先詢問，不要直接亂建資料夾。

---

## 步驟一：確認專案資料夾

請 Codex 先問使用者：

```text
這個專案要叫什麼名字？要放在 Google Drive、OneDrive，還是本機 Documents？
```

如果使用者不知道，建議：

```text
G:\我的雲端硬碟\2026Codex
```

Windows 範例：

```powershell
New-Item -ItemType Directory -Path "G:\我的雲端硬碟\2026Codex" -Force
```

建立後，請使用者在 Codex Desktop app 裡：

```text
專案 → 使用現有資料夾 → 選擇剛剛建立的資料夾
```

> [!important]
> Codex 的「聊天視窗」只是一般生成式 AI；要讓它真的讀寫檔案，請一定要進入「專案」並選定資料夾。

---

## 步驟二：盤點空資料夾

進入專案後，請 Codex 檢查資料夾內容：

```powershell
Get-ChildItem -Force
```

如果資料夾是空的，繼續初始化。

如果資料夾不是空的，請 Codex 先列出內容並詢問：

```text
這個資料夾已經有檔案。你要我在現有內容上補齊 Codex 專案設定，還是換一個空資料夾？
```

> [!warning]
> 不要在沒確認前刪除任何既有檔案。

---

## 步驟三：建立 README.md

`README.md` 是給人看的專案入口。

請建立：

```md
# <專案名稱>

## 專案目的

這個資料夾是我的 Codex 專案工作區，用來放置教材、程式、筆記或測試成果。

## 目前狀態

- 建立日期：YYYY-MM-DD
- 狀態：初始化

## 主要內容

- `AGENTS.md`：給 Codex 看的工作規則
- `README.md`：給人看的專案說明
- `.gitignore`：避免敏感資料或暫存檔被提交

## 使用方式

在 Codex Desktop app 中開啟此資料夾，讓 Codex 依照 `AGENTS.md` 的規則協助工作。
```

---

## 步驟四：建立 AGENTS.md

`AGENTS.md` 是給 Codex 看的專案規則，不是每日流水帳。

請建立：

```md
# <專案名稱> — AGENTS.md

## 專案定位

這是 Codex 專案工作區，用來讓 AI 協助讀取、建立、修改與整理本資料夾內的檔案。

## 使用者偏好

- 一律使用繁體中文
- 說明要簡潔，先講結論
- 涉及檔案操作時，回報完整檔案位置
- Windows 環境優先使用 PowerShell 指令

## 專案邊界

- 只修改本專案資料夾內與任務相關的檔案
- 修改既有檔案前先讀取內容
- 不要刪除使用者檔案，除非使用者明確要求
- 不要把 API Key、密碼、token 寫進任何會提交的檔案

## 檔案分工

- `AGENTS.md`：AI 固定規則、專案邊界、Do / Don't
- `README.md`：給使用者看的專案說明
- Obsidian 筆記：進度、踩坑、教學說明

## GitHub

- 尚未連接 GitHub repo
- 需要時再使用 [[Codex 懶人包/02-連接-GitHub]]

## Obsidian

- 若本專案需要第二大腦，使用 [[Codex 懶人包/03-建立第二大腦-Obsidian]]

## 開工 / 收工

- 開工時：先讀 `AGENTS.md`、`README.md`，檢查目前檔案狀態
- 收工時：整理變更摘要，必要時同步 Obsidian 與 GitHub
```

> [!important]
> `AGENTS.md` 寫固定規則，不寫每日進度。進度與踩坑應寫到 Obsidian 或 README 的狀態區。

---

## 步驟五：建立 .gitignore

請建立 `.gitignore`，避免敏感資料被提交。

```gitignore
# Secrets
.env
.env.*
*.key
*.pem
secrets/

# OpenAI / API keys
.openai.env

# Logs
*.log
logs/

# OS files
Thumbs.db
.DS_Store

# Temporary files
tmp/
temp/
*.tmp

# Build outputs
dist/
build/
node_modules/
__pycache__/
*.pyc

# Codex generated local scratch
.codex-tmp/
```

---

## 步驟六：驗證 Codex 可以讀寫檔案

請 Codex 建立測試檔：

```text
Codex 初始化測試.md
```

內容：

```md
# Codex 初始化測試

如果你看到這個檔案，代表 Codex 已經可以在本專案資料夾中建立檔案。

- 測試日期：YYYY-MM-DD
- 測試結果：成功
```

接著請 Codex 讀回檔案，確認內容存在。

驗證後詢問使用者：

```text
測試檔要保留，還是刪除？
```

> [!warning]
> 刪除測試檔前要先問，不要自動刪。

---

## 步驟七：可選，初始化 Git

如果使用者想之後接 GitHub，可以先初始化 Git。

請先問：

```text
要把這個資料夾初始化成 Git 專案嗎？現在只會建立本機 Git，不會 push 到 GitHub。
```

如果使用者同意：

```powershell
git init
git add README.md AGENTS.md .gitignore
git commit -m "初始化 Codex 專案工作區"
```

如果還沒設定 Git 使用者資訊，請引導：

```powershell
git config --global user.name "你的姓名"
git config --global user.email "你的email@example.com"
```

> [!tip]
> GitHub 上線請等下一集或使用 [[Codex 懶人包/02-連接-GitHub]]。本篇只處理本機專案初始化。

---

## 步驟八：檢查初學者必裝 Skills 與 Plugins

專案工作區建立好後，請 Codex 協助檢查目前有哪些 Skills 與 Plugins 已安裝或已啟用。

> [!important]
> 這一步不是要新手一次學會所有工具，而是先知道 Codex 未來可以接哪些能力。先檢查，缺的再補。

---

### 8.1 必裝 / 推薦清單

| 類型 | 名稱 | 用途 | 新手建議 |
|------|------|------|----------|
| Plugin | GitHub | 讀 repo、查 issue / PR、同步程式碼 | 必裝 |
| Plugin | Browser Use | 測試本機網頁、看畫面、截圖 | 必裝 |
| Plugin | Documents | 建立 / 修改 Word 講義、試卷 | 老師必裝 |
| Plugin | Presentations | 建立 / 修改 PowerPoint 簡報 | 老師必裝 |
| Plugin | Spreadsheets | Excel、CSV、題庫、成績表 | 老師必裝 |
| Skill | imagegen | 生圖、封面、教材圖、遊戲素材 | 推薦 |
| Skill | openai-docs | 查 OpenAI / Codex 官方文件 | 推薦 |
| Skill | skill-installer | 安裝其他 skills | 推薦 |
| Skill | skill-creator | 建立自己的 skills | 推薦 |
| MCP | Obsidian / MCPVault | 連接第二大腦筆記本 | 進階，EP04 再做 |

---

### 8.2 請 Codex 檢查本機 Skills

Windows 可檢查：

```powershell
Get-ChildItem -Force -Name "$env:USERPROFILE\.codex\skills"
Get-ChildItem -Force -Name "$env:USERPROFILE\.codex\skills\.system"
```

如果有看到以下項目，代表常用系統 skills 已存在：

```text
imagegen
openai-docs
skill-creator
skill-installer
plugin-creator
```

> [!tip]
> `plugin-creator` 是進階用，不是新手必學；但有裝著沒關係。

---

### 8.3 請 Codex 檢查 Plugins 是否啟用

請 Codex 查看設定檔：

```powershell
Get-Content -Encoding UTF8 -Path "$env:USERPROFILE\.codex\config.toml"
```

設定檔中若看到類似內容，代表 plugins 已啟用：

```toml
[plugins."browser-use@openai-bundled"]
enabled = true

[plugins."documents@openai-primary-runtime"]
enabled = true

[plugins."spreadsheets@openai-primary-runtime"]
enabled = true

[plugins."presentations@openai-primary-runtime"]
enabled = true

[plugins."github@openai-curated"]
enabled = true
```

老師新手建議至少確認：

- `browser-use`
- `documents`
- `presentations`
- `spreadsheets`
- `github`

---

### 8.4 GitHub 要分成兩種檢查

GitHub 有兩條路，不要混在一起：

| 類型 | 檢查方式 | 用途 |
|------|----------|------|
| GitHub CLI (`gh`) | `gh auth status` | commit / push / 建 repo / GitHub Pages |
| Codex GitHub connector | 請 Codex 列出 GitHub 帳號與可存取 repo | 對話中讀 repo / issue / PR |

檢查 GitHub CLI：

```powershell
gh auth status
```

成功時應看到：

```text
Logged in to github.com account <你的帳號>
Git operations protocol: https
Token scopes: ... repo ...
```

如果還沒登入，請使用 [[Codex 懶人包/02-連接-GitHub]]。

檢查 Codex GitHub connector 時，請使用者在 Codex 中輸入：

```text
幫我確認 GitHub connector 是否已連接，並列出我可以存取的 GitHub 帳號與幾個 repo。
```

如果出現：

```text
accounts: []
installations: []
repositories: []
```

代表 Codex Desktop 的 GitHub connector 還沒連上，下一集用 #02 懶人包處理。

---

### 8.5 缺少外掛時怎麼辦？

請使用者到 Codex Desktop app 設定中尋找：

```text
Settings / 設定
Plugins / Marketplace / Apps / Connectors
```

把以下項目啟用或安裝：

- GitHub
- Browser Use
- Documents
- Presentations
- Spreadsheets
- Gmail（需要才裝）
- Google Calendar（需要才裝）

> [!warning]
> 外掛不是越多越好。新手先把 GitHub、Browser Use、Documents、Presentations、Spreadsheets 裝好即可。Gmail / Calendar 涉及個人資料，確定要用再連。

---

### 8.6 Obsidian 先列進階，不在本集強裝

Obsidian / MCPVault 很重要，但建議放到後續集數。

原因：

- 每個人的 vault 位置不同
- 需要資料夾授權或 MCP 設定
- 需要解釋 `AGENTS.md` 與 Obsidian 駕駛艙分工

本集只要先讓觀眾知道：

```text
之後如果要讓 Codex 讀你的第二大腦，就會用 Codex 懶人包 #03。
```

## 新手常見問題

| 問題 | 解法 |
|------|------|
| 我在聊天視窗打指令，Codex 不能改檔 | 請進入「專案」並選定資料夾 |
| 不知道資料夾要放哪 | 先放 Google Drive 或 OneDrive，方便跨電腦同步 |
| Codex 說不能寫入 | 檢查是否選到正確工作資料夾，或是否需要授權 |
| AGENTS.md 要寫很多嗎 | 不用，先寫固定規則與邊界，之後再補 |
| 可以直接接 GitHub 嗎 | 可以，但建議先完成本篇，再做 #02 |
| 進度要寫 AGENTS.md 嗎 | 不建議，進度與踩坑寫 Obsidian 或 README |
| Skills 跟 Plugins 差在哪 | Skills 是工作流程；Plugins 是外部工具或能力包 |
| 一定要裝 Gmail / Calendar 嗎 | 不一定，需要整理信件或行程再裝 |
| Obsidian 要本集就裝嗎 | 不建議，本集先知道它是進階第二大腦，後面用 #03 做 |
